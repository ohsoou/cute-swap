# PRD.md — 오타쿠 굿즈 교환 서비스

---

## 1. 서비스 개요

**서비스명 (가칭)**: Kawaii Swap Market

**목표**
피규어, 문구류, CD 등 오타쿠 굿즈를 중심으로 한 **교환 중심 C2C 플랫폼**을 제공한다.
기본 거래 방식은 **물물교환**이며, 판매 기능은 추후 확장한다.

**핵심 가치**

* 교환 중심 구조
* 태그 기반 매칭
* 채팅 기반 협의
* 귀여운 UX/UI

---

## 2. 핵심 기능 요약

* 교환글 작성 (원하는 물품 태그 포함)
* 교환글 탐색 및 검색
* 게시글당 최대 3개 채팅 제한
* 채팅 기반 교환 협의 및 합의
* 찜, 알림, 신고 기능

---

## 3. 정보 구조 (IA) 및 라우팅

* `/login` → 로그인
* `/signup` → 회원가입

* `/posts/new` → 교환글 작성

* `/chat/[id]` → 채팅방

* `/mypage` → 마이페이지
* `/mypage/edit` → 프로필 수정

* `/notifications` → 알림
* `/favorites` → 찜 목록
* `/report` → 신고

### 공통

### [POST-002] `/posts/[id]` 교환글 상세

**목적**: 게시글 정보 확인 및 채팅 시작

**구성**

* 대표 이미지 갤러리
* 게시글 제목
* 작성자 정보
* 카테고리
* 물품 상태
* 상세 설명
* 원하는 태그 목록
* 채팅 가능 수
* 찜 버튼
* 신고 버튼
* 채팅 시작 버튼

**표시 규칙**

* 채팅 수 0~2개: 채팅 가능
* 채팅 수 3개: 채팅 마감
* 교환 진행중: 별도 배지 표시
* 완료: 읽기 전용 처리

**동작**

* 채팅 시작 시 채팅 가능 수를 서버에서 검증한다.
* 찜 클릭 시 상태를 토글한다.
* 신고 클릭 시 `/report`로 이동하거나 신고 모달을 연다.
* 작성자 프로필 진입 가능

**예외**

* 게시글 삭제됨
* 채팅 3개 초과 시도
* 비로그인 상태에서 채팅 시작 시 로그인 유도

## 5. 핵심 비즈니스 로직

### 채팅 제한

* 게시글당 최대 3개 채팅만 허용
* 초과 시 채팅 생성 불가

### 채팅 생성 조건

* 로그인 상태
* 채팅 수 < 3
* 동일 사용자 중복 채팅 제한 여부는 정책으로 확정 필요

### 상태 흐름

#### 게시글

* 게시중 → 채팅 마감 → 진행중 → 완료

#### 채팅

* 진행중 → 합의완료 → 종료

## 7. 기술 스택 및 아키텍처

### 프론트엔드 (Next.js)

* Next.js App Router
* SSR / SSG 혼합 사용
* Tailwind CSS
* 상태 관리: React Query, Zustand

### 백엔드 (Supabase)

* Supabase Auth → 인증
* Supabase Database (PostgreSQL)
* Supabase Storage → 이미지 업로드
* Supabase Realtime → 채팅 기능

### 구조

* Next.js는 UI와 일부 서버 로직(BFF 역할)을 담당한다.
* Supabase는 DB, Auth, Storage, Realtime을 담당한다.
* 클라이언트는 단순 조회는 Supabase SDK를 직접 사용하고, 민감한 검증은 서버 경유로 처리한다.

## 8. 주요 기술 설계

### 인증

* Supabase Auth 사용
* 이메일 기반 로그인/회원가입
* 세션은 쿠키 기반으로 유지

### 채팅

* Supabase Realtime 기반
* ChatMessage 테이블 구독
* 채팅방 단위 채널 구독

### 채팅 제한

* 채팅 생성 시 트랜잭션 처리
* 조건: `chatCount < 3`
* 실패 시 에러 반환

### 이미지 업로드

* Supabase Storage 사용
* 업로드 후 public URL 저장

### 태그 시스템

* Tag 테이블 + 매핑 테이블 사용
* 자동완성은 LIKE 검색 기반으로 시작 가능
